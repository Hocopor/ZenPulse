# ZenPulse Live Plan

## 1. Purpose

Этот документ является живым планом реализации ZenPulse.

Он:
- не заменяет `docs/`;
- не заменяет `coordination/Implementation_Spec.md`;
- фиксирует текущий статус проекта;
- задаёт порядок работ;
- должен обновляться после каждого завершённого шага.

Перед началом любой работы порядок действий всегда такой:
1. Открыть этот файл.
2. Найти активный этап.
3. Открыть `coordination/Implementation_Spec.md`.
4. Перейти в указанные там документы `docs/`.
5. Выполнить только текущий этап.
6. Обновить этот файл.

## 2. Status Legend

## 3. Project Guardrails

Во всех этапах действуют неизменные правила:
- единственный source of truth находится в `docs/`;
- нельзя добавлять новые экраны;
- нельзя расширять MVP;
- нельзя подключать backend, real AI или real billing;
- нельзя писать код без проверки релевантных документов;
- после каждого этапа нужно отметить прогресс и коротко зафиксировать результат.

## 6. Live Update Rules

После каждого завершённого шага обязательно:
- перевести статус соответствующего этапа;
- коротко записать, что именно завершено;
- если найден новый риск, добавить его в раздел рисков;
- если появился блокер, отметить этап как `[blocked]` и зафиксировать причину;
- если этап декомпозируется на подшаги, добавить их прямо в этот файл.

Нельзя:
- держать план формально “как есть” после изменений;
- оставлять активными сразу несколько крупных этапов без явной причины;
- отмечать этап завершённым без проверки по документации.

## 7. Risks And Control Points

Текущие контрольные риски:
- риск дрейфа требований при попытке “улучшить” MVP сверх `docs/`;
- риск появления лишнего flow вокруг free-карточек;
- риск неправильного gate для вкладки `Настрой дня`;
- риск неверного поведения Paywall после уже активной подписки;
- риск layout-поломок на маленькой высоте экрана;
- риск длинных AI-текстов, ломающих карточку результата;
- риск размазывания `isSubscribed` по нескольким несогласованным флагам.

Контрольные точки:
- после Этапа 2 проверить single source of truth;
- после Этапа 3 проверить всю матрицу переходов;
- после Этапа 6 проверить 2 free / 4 premium;
- после Этапа 7 проверить active Premium state;
- после Этапа 8 проверить все mood-сценарии;
- после Этапа 10 провести финальную сверку с `docs/`.
